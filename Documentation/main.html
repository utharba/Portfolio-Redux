Git 101

Version Control

Version control allows you to keep track of changes made to a file and allows past versions to be recovered easily. Git is a distributed version control system that gives each user a full copy of a centralized repository. This means that in the event of an issue with the central copy (or one of the users' copies), it can be easily reverted or checked for the time and user who made those changes.

Git Status

git status will return all the basic information you need to know about your branch file, including:

  • the current branch
  • whether that branch is up to date with the master copy
  • any untracked files
  • uncommitted changes
  • the commands to track and commit any changed (in case you don't remember)

Git Init

git init will initialize the current directory as a local Git repository. You can then use git remote add origin <"remote repository url"> to set the local repository as the source for the remote repository.

You can check if your source is set up correctly with the command git remote -v and checking that it returns the remote repository's url.

Add File

add file <"filename"> will add the named file to your local staging area.

Commit

git commit will commit your staged changes to your local copy of the repository.

Push

git push <"local_branch_name"> <"repository name"> will push your local changes to the central repository.

Command Line

Some generally useful commands:

  • cd <"directory_name"> - change the current directory to the child or parent "directory_name"
  • pwd - path of working directory
  • ls - lists directory contents

Github

Github is the primary remote repository used with Git. To create a new repository, simply go to your account's repository tab and click new repository. Then once you've configured it to your liking, hit create and then use the new repository's url in the git remote add origin command.
